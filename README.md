## Descrição do problema
A ferramenta proposta tem como objetivo analisar a centralização de conhecimento em diversos repositórios no GitHub e identificar potenciais problemas de manutenção gerados por baixo Truck Factor. 

## Tecnologias utilizadas
As tecnologias utilizadas serão: <b>[GitHub Search tool (GHS)](https://seart-ghs.si.usi.ch/)</b> para seleção de repositórios que serão analisados, <b>[GitEvo](https://github.com/andrehora/gitevo)</b> e <b>[pydriller](https://github.com/ishepard/pydriller)</b> para obtenção de métricas e <b>[typer](https://github.com/fastapi/typer)</b> para criação da ferramenta de linha de comando.

## Artefatos analisados
Os artefatos analisados incluem o histórico de commits para mapear a autoria, os arquivos de código fonte para identificar a propriedade de módulos específicos e os logs de modificação para mensurar o volume de contribuição técnica e complexidade acrescentada por desenvolvedor.
